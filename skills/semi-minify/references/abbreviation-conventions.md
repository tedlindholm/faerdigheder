# Abbreviation conventions

## Rule

Keep consonants that carry the word's identity; drop vowels and trailing syllables. Be consistent — the same word always maps to the same short form. camelCase is preserved.

## Common mappings

| Word | Short |
|------|-------|
| export | `e` |
| element | `el` |
| children | `ch` |
| remote | `rmt` |
| config / configuration | `cfg` |
| context | `ctx` |
| reference | `ref` |
| module | `mod` |
| dependency | `dep` |
| transform | `xfm` |
| manager | `mgr` |
| instance | `inst` |
| register / registry | `reg` |
| initialise | `init` |
| handler | `hnd` |
| options | `opts` |
| parameter | `prm` |
| normalise | `norm` |
| validate | `val` |
| compute / calculate | `calc` |
| local | `loc` |
| shared | `shr` |
| consumer | `cns` |
| marker | `mrk` |
| binding | `bnd` |
| factory | `fac` |
| graph | `gr` |

## Constraints

- Do not produce an abbreviation that already exists in the file under a different meaning.
- Preserve boolean prefixes (`is`, `has`, `can`, `should`) intact.
- Preserve acronyms (`DOM`, `API`, `URL`) verbatim.
